# Analytics

## Purpose

The analytics experience should help hosts and property managers understand guest communication, operations, and business performance.

## Primary Content

- Conversation volume.
- Response times.
- AI resolution and escalation trends.
- Service request volume and resolution time.
- Property-level activity.
- Payment or revenue summaries where applicable.

## UX Goals

- Make metrics easy to interpret.
- Show trends and exceptions, not just totals.
- Keep filters clear and company-scoped.
- Link metrics to operational details where action is needed.

## States

- Empty analytics state.
- Normal reporting.
- Filtered reporting.
- Loading state.
- Partial data state.
- Data unavailable state.

## Future Considerations

Define metric names and calculations in product and API documentation so dashboards remain consistent across teams.
