# Dashboard

## Purpose

The dashboard should give hosts and property managers a fast overview of portfolio health, guest activity, service requests, and operational alerts.

## Primary Content

- Active properties.
- Open guest conversations.
- Pending service requests.
- Message response trends.
- Payment or billing alerts where applicable.
- AI concierge activity and escalations.

## UX Goals

- Prioritize actionable operational information.
- Keep metrics scannable and clearly labeled.
- Avoid decorative layouts that hide urgent work.
- Provide clear navigation to detailed workflows.

## States

- Empty company or first-time setup.
- Normal operating state.
- High activity or urgent alerts.
- Loading and partial data states.
- Error state for unavailable metrics.

## Future Considerations

Dashboard widgets should become configurable only after the core operational metrics are validated with users.
