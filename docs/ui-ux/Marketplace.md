# Marketplace

## Purpose

The marketplace experience will help property managers discover, coordinate, and manage service providers for property operations.

## Primary Content

- Service provider profiles.
- Service categories.
- Availability and coverage areas.
- Service request creation.
- Assignment and status tracking.
- Provider contact details.

## UX Goals

- Help operators find suitable providers quickly.
- Make service request status obvious.
- Reduce manual coordination through clear workflows.
- Keep guest-facing and internal operational notes distinct.

## States

- Provider list.
- Provider detail.
- Create service request.
- Assigned request.
- In-progress request.
- Completed request.
- Failed or cancelled request.

## Future Considerations

Marketplace trust, ratings, pricing, and payment workflows should be designed after provider operations are validated.
