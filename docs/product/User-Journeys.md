# User Journeys

## Purpose

This document captures important user journeys for StayFlow AI and should guide feature design and acceptance criteria.

## Host Onboards a Property

1. Host creates or joins a company account.
2. Host adds property details.
3. Host adds amenities, house rules, emergency contacts, recommendations, and knowledge articles.
4. Host verifies the property profile.
5. Property becomes available for concierge workflows.

## Guest Asks a Question

1. Guest sends a WhatsApp message.
2. StayFlow AI identifies the company, property, guest, and conversation context.
3. The AI concierge retrieves relevant property knowledge.
4. A response is generated or the request is escalated.
5. The guest receives a timely reply.

## Guest Requests Help

1. Guest reports an issue or asks for a service.
2. The system creates or updates a service request.
3. The request is assigned or escalated.
4. The guest receives status updates.
5. The issue is resolved and recorded.

## Manager Reviews Operations

1. Property manager opens operational dashboards or reports.
2. Manager reviews guest conversations, service requests, payments, and escalations.
3. Manager identifies unresolved issues.
4. Manager updates property knowledge or operating procedures.

## Journey Documentation Rules

Each journey should include actors, entry points, success criteria, failure paths, and required data.
