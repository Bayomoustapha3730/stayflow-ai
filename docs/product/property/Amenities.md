# Amenities

## Purpose

Amenities describe features and conveniences available at a property. They help guests understand what is included and help the AI concierge answer common questions.

## Examples

- Wi-Fi.
- Parking.
- Pool.
- Air conditioning.
- Backup power.
- Kitchen.
- Workspace.
- Laundry.
- Security.

## Product Requirements

- Hosts should be able to add multiple amenities per property.
- Each amenity should have a clear name.
- Optional descriptions should clarify usage, limits, or location.
- Amenities should be available to property detail views and AI concierge context.
- Inactive or deleted amenities should not be used in guest-facing answers.

## Content Guidance

Amenity descriptions should be specific enough to answer guest questions. For example, "Wi-Fi available" is less helpful than "Wi-Fi network details are shared after check-in; backup router is available in the living room."

## Future Enhancements

- Amenity categories.
- Icons for common amenities.
- Guest-visible versus internal-only flags.
- Readiness checks for missing descriptions on important amenities.
