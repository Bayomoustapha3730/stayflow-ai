# Property Overview

## Purpose

The property domain is the foundation for StayFlow AI guest concierge workflows. It stores the operational and guest-facing information required to answer questions, support stays, and coordinate property-specific services.

## Product Scope

A property profile should capture:

- Property identity and location.
- Guest-facing description.
- Amenities.
- House rules.
- Emergency contacts.
- Local recommendations.
- AI knowledge base articles.
- Active or inactive status.

## User Value

For hosts and property managers, property profiles reduce repetitive guest communication and create a single source of truth. For guests, property profiles power fast, accurate answers about check-in, house rules, safety, local services, and stay expectations.

## Primary Users

- Airbnb hosts.
- Property managers.
- Operations team members.
- Guest support agents.
- AI concierge workflows.

## Product Principles

- Property setup should be structured but not burdensome.
- Guest-facing information should be easy to review and update.
- AI-facing knowledge should be accurate, specific, and auditable.
- Company isolation must be preserved for all property data.
