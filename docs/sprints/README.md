# Sprint Documentation

This folder tracks planning and delivery notes for StayFlow AI sprints.

Use it for:

- Sprint goals and scope.
- Planned deliverables and acceptance criteria.
- Implementation notes and release summaries.
- Backlog slices and follow-up items.
- Risks, blockers, and decisions discovered during delivery.

Sprint documents should be dated and written so future contributors can understand what changed, why it changed, and what remains open.
