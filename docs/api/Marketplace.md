# Marketplace API

## Purpose

Marketplace APIs will support service provider discovery, service offerings, operational requests, and vendor coordination for property managers.

## Planned Endpoint Areas

- Service provider listing and search.
- Service provider profile management.
- Service category and availability metadata.
- Service request creation and assignment.
- Service request status tracking.

## Data Considerations

Marketplace workflows should remain company-scoped unless a provider is intentionally shared across companies. Service request operations should preserve audit history and status transitions.

## Integration Notes

Marketplace APIs may later integrate with WhatsApp notifications, payment workflows, and background services for dispatch and follow-up.

Add routes, DTOs, lifecycle states, and examples when marketplace modules are implemented.
